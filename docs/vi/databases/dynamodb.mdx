# Kết nối Amazon DynamoDB

TablePro hỗ trợ Amazon DynamoDB, cơ sở dữ liệu NoSQL key-value và document được quản lý hoàn toàn bởi AWS. Kết nối bằng IAM credentials, AWS profiles hoặc SSO. Duyệt bảng, chạy truy vấn PartiQL, kiểm tra index GSI/LSI và chỉnh sửa item trực tiếp trong data grid.

## Cài đặt Plugin

Driver DynamoDB có sẵn dưới dạng plugin tải về. Khi bạn chọn DynamoDB trong form kết nối, TablePro sẽ tự động nhắc bạn cài đặt. Bạn cũng có thể cài đặt thủ công:

1. Mở **Cài đặt** > **Plugin** > **Duyệt**
2. Tìm **DynamoDB Driver** và nhấn **Cài đặt**
3. Plugin được tải và kích hoạt ngay lập tức, không cần khởi động lại

## Phương thức Xác thực

DynamoDB hỗ trợ ba phương thức xác thực:

### Access Key + Secret Key

Sử dụng trực tiếp thông tin đăng nhập IAM user. Phù hợp nhất cho phát triển và thiết lập đơn giản.

| Trường | Mô tả |
|--------|-------|
| **Access Key ID** | IAM access key (bắt đầu bằng `AKIA`) |
| **Secret Access Key** | IAM secret key |
| **Session Token** | Tùy chọn. Bắt buộc cho temporary credentials từ STS |
| **Region** | Vùng AWS (ví dụ: `us-east-1`) |

### AWS Profile

Đọc credentials từ `~/.aws/credentials`. Phù hợp nhất khi quản lý nhiều tài khoản AWS.

| Trường | Mô tả |
|--------|-------|
| **Profile Name** | Tên profile trong `~/.aws/credentials` (mặc định: `default`) |
| **Region** | Vùng AWS |

```ini
# ~/.aws/credentials
[default]
aws_access_key_id = AKIA...
aws_secret_access_key = wJalr...

[staging]
aws_access_key_id = AKIA...
aws_secret_access_key = ...
aws_session_token = ...
```

### AWS SSO

Sử dụng SSO credentials đã cache trong `~/.aws/cli/cache/`. Chạy `aws sso login` trước.

| Trường | Mô tả |
|--------|-------|
| **Region** | Vùng AWS |

```bash
# Đăng nhập qua SSO trước khi kết nối
aws sso login --profile my-sso-profile
```

<Warning>
SSO credentials có thời hạn. Nếu kết nối thất bại với lỗi xác thực, hãy chạy lại `aws sso login`.
</Warning>

## Cài đặt Kết nối

### Trường bắt buộc

| Trường | Mô tả |
|--------|-------|
| **Name** | Tên kết nối |
| **Auth Method** | Access Key + Secret Key, AWS Profile, hoặc AWS SSO |
| **Region** | Vùng AWS nơi các bảng của bạn được lưu trữ |

### Trường tùy chọn

| Trường | Mô tả |
|--------|-------|
| **Custom Endpoint** | Ghi đè URL endpoint DynamoDB (cho DynamoDB Local) |

## DynamoDB Local

Để phát triển cục bộ, sử dụng [DynamoDB Local](https://docs.aws.amazon.com/amazondynamodb/latest/developerguide/DynamoDBLocal.html):

```bash
# Khởi động DynamoDB Local với Docker
docker run -p 8000:8000 amazon/dynamodb-local
```

Sau đó cấu hình:
- **Auth Method**: Access Key + Secret Key
- **Access Key ID**: `fakeAccessKey` (bất kỳ giá trị không rỗng)
- **Secret Access Key**: `fakeSecretKey` (bất kỳ giá trị không rỗng)
- **Custom Endpoint**: `http://localhost:8000`

<Tip>
DynamoDB Local không xác thực credentials, nhưng các trường phải có giá trị.
</Tip>

## Tính năng

### Duyệt Bảng

Sau khi kết nối, sidebar liệt kê tất cả các bảng DynamoDB. Cho mỗi bảng, TablePro hiển thị:

- **Data**: Item hiển thị dưới dạng lưới với tất cả thuộc tính được phát hiện
- **Structure**: Key schema với kiểu DynamoDB (S, N, B)
- **Indexes**: Primary key, Global Secondary Indexes (GSI) và Local Secondary Indexes (LSI)
- **DDL**: Mô tả bảng bao gồm key schema, billing mode, throughput, indexes, số item và kích thước

### Phát hiện Cột Schemaless

Bảng DynamoDB là schemaless: các item khác nhau có thể có thuộc tính khác nhau. TablePro lấy mẫu item để phát hiện tất cả tên thuộc tính và suy luận kiểu cột. Thuộc tính key (partition key, sort key) luôn xuất hiện đầu tiên.

### Truy vấn PartiQL

Viết và thực thi truy vấn [PartiQL](https://docs.aws.amazon.com/amazondynamodb/latest/developerguide/ql-reference.html) trong editor:

```sql
-- Truy vấn item
SELECT * FROM "Users" WHERE userId = 'user123'

-- Thêm item
INSERT INTO "Users" VALUE {'userId': 'user456', 'name': 'Alice', 'age': 30}

-- Cập nhật item
UPDATE "Users" SET name = 'Bob' WHERE userId = 'user123'

-- Xóa item
DELETE FROM "Users" WHERE userId = 'user123'
```

<Tip>
Tên bảng trong PartiQL phải được đặt trong dấu ngoặc kép: `"TableName"`. Giá trị chuỗi dùng dấu nháy đơn: `'value'`.
</Tip>

### Kiểu Dữ liệu

| Kiểu DynamoDB | Hiển thị |
|----------------|----------|
| **S** (String) | Văn bản |
| **N** (Number) | Giá trị số |
| **B** (Binary) | Chuỗi base64 |
| **BOOL** | `true` / `false` |
| **NULL** | `NULL` |
| **L** (List) | Mảng JSON |
| **M** (Map) | Object JSON |
| **SS** (String Set) | Mảng JSON chuỗi |
| **NS** (Number Set) | Mảng JSON số |
| **BS** (Binary Set) | Mảng JSON base64 |

### Chỉnh sửa Dữ liệu

Chỉnh sửa giá trị ô, thêm hàng và xóa hàng trực tiếp trong data grid. Thay đổi tạo ra câu lệnh PartiQL tự động.

<Note>
Thuộc tính key (partition key và sort key) không thể thay đổi. Để thay đổi giá trị key, xóa item và thêm item mới.
</Note>

### Tối ưu Truy vấn

Khi lọc theo partition key với điều kiện bằng, TablePro tự động sử dụng Query API thay vì Scan. Query nhanh hơn đáng kể và tiêu tốn ít read capacity hơn.

### Dung lượng và Số liệu

Metadata bảng hiển thị billing mode, số item ước tính, kích thước bảng và chi tiết index.

### Xuất Dữ liệu

Xuất kết quả truy vấn hoặc dữ liệu bảng sang CSV, JSON, SQL và các định dạng khác.

## Xử lý Sự cố

### Xác thực Thất bại

**Triệu chứng**: "Authentication failed" hoặc "UnrecognizedClientException"

**Giải pháp**:

1. Kiểm tra Access Key ID và Secret Access Key
2. Kiểm tra credentials chưa hết hạn
3. Với AWS Profile, xác nhận profile tồn tại trong `~/.aws/credentials`
4. Với SSO, chạy `aws sso login` để làm mới credentials

### Bị Từ chối Quyền truy cập

**Triệu chứng**: "AccessDeniedException"

**Giải pháp**:

1. Kiểm tra quyền IAM - user/role cần quyền truy cập DynamoDB
2. Xác minh resource ARN nếu dùng quyền cấp resource

### Sai Vùng

**Triệu chứng**: Không tìm thấy bảng hoặc danh sách bảng trống

**Giải pháp**:

1. Xác nhận đã chọn đúng vùng AWS
2. Bảng DynamoDB là riêng theo vùng - bảng ở `us-east-1` không hiển thị từ `eu-west-1`

## Hạn chế

- **Không có kết nối liên tục.** DynamoDB dựa trên HTTP. Mỗi thao tác là một API request độc lập.
- **Không có SSH hoặc SSL.** DynamoDB sử dụng HTTPS qua URLSession với SigV4 request signing.
- **Không chỉnh sửa schema.** DynamoDB không hỗ trợ ALTER TABLE.
- **Số item ước tính.** Số item được cập nhật bởi DynamoDB mỗi ~6 giờ.
- **Phân trang dựa trên cursor.** DynamoDB sử dụng continuation token nội bộ.
- **Không hỗ trợ DAX.** DynamoDB Accelerator không được hỗ trợ.
