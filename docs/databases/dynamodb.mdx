# Amazon DynamoDB Connections

TablePro supports Amazon DynamoDB, AWS's fully managed NoSQL key-value and document database. Connect using IAM credentials, AWS profiles, or SSO. Browse tables, run PartiQL queries, inspect GSI/LSI indexes, and edit items directly in the data grid.

## Install Plugin

The DynamoDB driver is available as a downloadable plugin. When you select DynamoDB in the connection form, TablePro will prompt you to install it automatically. You can also install it manually:

1. Open **Settings** > **Plugins** > **Browse**
2. Find **DynamoDB Driver** and click **Install**
3. The plugin downloads and loads immediately - no restart needed

## Authentication Methods

DynamoDB supports three authentication methods:

### Access Key + Secret Key

Use IAM user credentials directly. Best for development and simple setups.

| Field | Description |
|-------|-------------|
| **Access Key ID** | IAM access key (starts with `AKIA`) |
| **Secret Access Key** | IAM secret key |
| **Session Token** | Optional. Required for temporary credentials from STS |
| **Region** | AWS region (e.g., `us-east-1`) |

### AWS Profile

Read credentials from `~/.aws/credentials`. Best when you manage multiple AWS accounts.

| Field | Description |
|-------|-------------|
| **Profile Name** | Profile name in `~/.aws/credentials` (default: `default`) |
| **Region** | AWS region |

```ini
# ~/.aws/credentials
[default]
aws_access_key_id = AKIA...
aws_secret_access_key = wJalr...

[staging]
aws_access_key_id = AKIA...
aws_secret_access_key = ...
aws_session_token = ...
```

### AWS SSO

Use cached SSO credentials from `~/.aws/cli/cache/`. Run `aws sso login` first.

| Field | Description |
|-------|-------------|
| **Region** | AWS region |

```bash
# Login via SSO before connecting
aws sso login --profile my-sso-profile
```

<Warning>
SSO credentials expire. If your connection fails with an auth error, run `aws sso login` again.
</Warning>

## Connection Settings

### Required Fields

| Field | Description |
|-------|-------------|
| **Name** | Connection identifier |
| **Auth Method** | Access Key + Secret Key, AWS Profile, or AWS SSO |
| **Region** | AWS region where your tables are located |

### Optional Fields

| Field | Description |
|-------|-------------|
| **Custom Endpoint** | Override the DynamoDB endpoint URL (for DynamoDB Local) |

## DynamoDB Local

For local development, use [DynamoDB Local](https://docs.aws.amazon.com/amazondynamodb/latest/developerguide/DynamoDBLocal.html):

```bash
# Start DynamoDB Local with Docker
docker run -p 8000:8000 amazon/dynamodb-local
```

Then set:
- **Auth Method**: Access Key + Secret Key
- **Access Key ID**: `fakeAccessKey` (any non-empty value)
- **Secret Access Key**: `fakeSecretKey` (any non-empty value)
- **Custom Endpoint**: `http://localhost:8000`

<Tip>
DynamoDB Local does not validate credentials, but the fields must be non-empty.
</Tip>

## Features

### Table Browsing

After connecting, the sidebar lists all DynamoDB tables. For each table, TablePro shows:

- **Data**: Items displayed in a tabular grid with all discovered attributes as columns
- **Structure**: Key schema attributes with their DynamoDB types (S, N, B)
- **Indexes**: Primary key, Global Secondary Indexes (GSI), and Local Secondary Indexes (LSI) with projection types
- **DDL**: Table description including key schema, billing mode, throughput, indexes, item count, and table size

### Schemaless Column Discovery

DynamoDB tables are schemaless: different items can have different attributes. TablePro samples items to discover all attribute names and infers column types by majority vote. Key attributes (partition key, sort key) always appear as the first columns.

### PartiQL Queries

Write and execute [PartiQL](https://docs.aws.amazon.com/amazondynamodb/latest/developerguide/ql-reference.html) queries in the editor:

```sql
-- Select items
SELECT * FROM "Users" WHERE userId = 'user123'

-- Insert an item
INSERT INTO "Users" VALUE {'userId': 'user456', 'name': 'Alice', 'age': 30}

-- Update an item
UPDATE "Users" SET name = 'Bob' WHERE userId = 'user123'

-- Delete an item
DELETE FROM "Users" WHERE userId = 'user123'

-- Query with conditions
SELECT * FROM "Orders"
WHERE customerId = 'cust001' AND orderDate BETWEEN '2024-01-01' AND '2024-12-31'
```

<Tip>
Table names in PartiQL must be quoted with double quotes: `"TableName"`. String values use single quotes: `'value'`.
</Tip>

### Data Types

DynamoDB attribute values display with type-aware formatting:

| DynamoDB Type | Display |
|---------------|---------|
| **S** (String) | Plain text |
| **N** (Number) | Numeric value |
| **B** (Binary) | Base64-encoded string |
| **BOOL** | `true` / `false` |
| **NULL** | `NULL` |
| **L** (List) | JSON array |
| **M** (Map) | JSON object |
| **SS** (String Set) | JSON array of strings |
| **NS** (Number Set) | JSON array of numbers |
| **BS** (Binary Set) | JSON array of base64 strings |

### Data Editing

Edit cell values, insert rows, and delete rows directly in the data grid. Changes generate PartiQL statements:

- **Insert**: `INSERT INTO "Table" VALUE { ... }`
- **Update**: `UPDATE "Table" SET attr = 'val' WHERE pk = 'pkVal'`
- **Delete**: `DELETE FROM "Table" WHERE pk = 'pkVal'`

<Note>
Key attributes (partition key and sort key) cannot be modified. To change a key value, delete the item and insert a new one.
</Note>

### Smart Query Optimization

When filtering by partition key with an equality condition, TablePro automatically uses the DynamoDB Query API instead of Scan. Query is significantly faster and consumes less read capacity.

### Capacity and Metrics

Table metadata displays:

- **Billing mode**: PAY_PER_REQUEST or PROVISIONED with RCU/WCU values
- **Item count**: Approximate number of items (updated by DynamoDB every ~6 hours)
- **Table size**: Total storage used
- **Index details**: GSI/LSI key schemas, projection types, and status

### Export

Export query results or table data to CSV, JSON, SQL, and other formats.

## IAM Permissions

The IAM user or role needs these permissions at minimum:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "dynamodb:ListTables",
        "dynamodb:DescribeTable",
        "dynamodb:Scan",
        "dynamodb:Query",
        "dynamodb:GetItem",
        "dynamodb:PutItem",
        "dynamodb:UpdateItem",
        "dynamodb:DeleteItem",
        "dynamodb:PartiQLSelect",
        "dynamodb:PartiQLInsert",
        "dynamodb:PartiQLUpdate",
        "dynamodb:PartiQLDelete"
      ],
      "Resource": "*"
    }
  ]
}
```

For read-only access, remove the write actions (`PutItem`, `UpdateItem`, `DeleteItem`, `PartiQLInsert`, `PartiQLUpdate`, `PartiQLDelete`).

## Troubleshooting

### Authentication Failed

**Symptoms**: "Authentication failed" or "UnrecognizedClientException"

**Solutions**:

1. Verify your Access Key ID and Secret Access Key are correct
2. Check that the credentials have not expired or been rotated
3. For AWS Profile, verify the profile exists in `~/.aws/credentials`
4. For SSO, run `aws sso login` to refresh credentials

### Access Denied

**Symptoms**: "AccessDeniedException" on specific operations

**Solutions**:

1. Check IAM permissions - the user/role needs DynamoDB access
2. Verify the resource ARN if using resource-level permissions
3. Check for restrictive SCP or permission boundaries

### Region Mismatch

**Symptoms**: Tables not found or empty table list

**Solutions**:

1. Verify you selected the correct AWS region
2. DynamoDB tables are region-specific - a table in `us-east-1` is not visible from `eu-west-1`

### Throughput Exceeded

**Symptoms**: "ProvisionedThroughputExceededException"

**Solutions**:

1. The table's provisioned read/write capacity has been exceeded
2. Wait and retry - the plugin does not include automatic retry
3. Consider switching to on-demand billing mode
4. Use filters to reduce the number of items scanned

## Known Limitations

- **No persistent connections.** DynamoDB is HTTP-based. Each operation is an independent API request.
- **No SSH or SSL configuration.** DynamoDB uses HTTPS transport via URLSession with custom SigV4 request signing. No custom SSL certificates or SSH tunnels are needed.
- **Schema editing unavailable.** DynamoDB does not support ALTER TABLE. Table structure is defined at creation time.
- **Approximate item counts.** The item count shown in table metadata is approximate and updated by DynamoDB every ~6 hours.
- **Cursor-based pagination.** DynamoDB uses continuation tokens internally. Jumping to an arbitrary page requires scanning through all preceding items.
- **Transactions not exposed.** DynamoDB transactions (TransactWriteItems/TransactGetItems) are not exposed in the UI. Use the AWS Console or CLI for transactional operations.
- **DAX unsupported.** DynamoDB Accelerator (DAX) in-memory caching is not supported.
- **Global Tables management unavailable.** Multi-region Global Tables configuration is not available through the plugin.
