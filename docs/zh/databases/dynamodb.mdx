# Amazon DynamoDB 连接

TablePro 支持 Amazon DynamoDB，AWS 的全托管 NoSQL 键值和文档数据库。使用 IAM 凭证、AWS 配置文件或 SSO 进行连接。浏览表、运行 PartiQL 查询、检查 GSI/LSI 索引，并直接在数据网格中编辑项目。

## 安装插件

DynamoDB 驱动以可下载插件的形式提供。当您在连接表单中选择 DynamoDB 时，TablePro 会自动提示您安装。您也可以手动安装：

1. 打开 **设置** > **插件** > **浏览**
2. 找到 **DynamoDB Driver** 并点击 **安装**
3. 插件立即下载并加载，无需重启

## 快速设置

<Steps>
  <Step title="打开连接表单">
    在欢迎界面点击 **New Connection**，或通过 **File** > **New Connection**
  </Step>
  <Step title="选择 Amazon DynamoDB">
    从数据库类型选择器中选择 **Amazon DynamoDB**
  </Step>
  <Step title="配置认证">
    选择认证方式并输入 AWS 凭证（见下文）
  </Step>
  <Step title="测试并连接">
    点击 **Test Connection**，然后点击 **Create**
  </Step>
</Steps>

## 认证方式

DynamoDB 支持三种认证方式：

### Access Key + Secret Key

直接使用 IAM 用户凭证。最适合开发和简单设置。

| 字段 | 描述 |
|------|------|
| **Access Key ID** | IAM access key（以 `AKIA` 开头） |
| **Secret Access Key** | IAM secret key |
| **Session Token** | 可选。STS 临时凭证需要 |
| **Region** | AWS 区域（例如 `us-east-1`） |

### AWS Profile

从 `~/.aws/credentials` 读取凭证。最适合管理多个 AWS 账户。

| 字段 | 描述 |
|------|------|
| **Profile Name** | `~/.aws/credentials` 中的配置文件名（默认：`default`） |
| **Region** | AWS 区域 |

```ini
# ~/.aws/credentials
[default]
aws_access_key_id = AKIA...
aws_secret_access_key = wJalr...

[staging]
aws_access_key_id = AKIA...
aws_secret_access_key = ...
aws_session_token = ...
```

### AWS SSO

使用 `~/.aws/cli/cache/` 中缓存的 SSO 凭证。需先运行 `aws sso login`。

| 字段 | 描述 |
|------|------|
| **Region** | AWS 区域 |

```bash
# 连接前通过 SSO 登录
aws sso login --profile my-sso-profile
```

<Warning>
SSO 凭证会过期。如果连接因认证错误失败，请重新运行 `aws sso login`。
</Warning>

## 连接设置

### 必填字段

| 字段 | 描述 |
|------|------|
| **Name** | 连接名称 |
| **Auth Method** | Access Key + Secret Key、AWS Profile 或 AWS SSO |
| **Region** | 表所在的 AWS 区域 |

### 可选字段

| 字段 | 描述 |
|------|------|
| **Custom Endpoint** | 覆盖 DynamoDB 端点 URL（用于 DynamoDB Local） |

## DynamoDB Local

用于本地开发，使用 [DynamoDB Local](https://docs.aws.amazon.com/amazondynamodb/latest/developerguide/DynamoDBLocal.html)：

```bash
# 使用 Docker 启动 DynamoDB Local
docker run -p 8000:8000 amazon/dynamodb-local
```

然后设置：
- **Auth Method**: Access Key + Secret Key
- **Access Key ID**: `fakeAccessKey`（任意非空值）
- **Secret Access Key**: `fakeSecretKey`（任意非空值）
- **Custom Endpoint**: `http://localhost:8000`

<Tip>
DynamoDB Local 不验证凭证，但字段必须有值。
</Tip>

## 功能

### 表浏览

连接后，侧边栏列出所有 DynamoDB 表。对于每个表，TablePro 显示：

- **Data**：项目以表格网格形式显示，包含所有发现的属性列
- **Structure**：键模式属性及其 DynamoDB 类型（S、N、B）
- **Indexes**：主键、全局二级索引（GSI）和本地二级索引（LSI）及投影类型
- **DDL**：表描述，包括键模式、计费模式、吞吐量、索引、项目数和表大小

### 无模式列发现

DynamoDB 表是无模式的：不同的项目可以有不同的属性。TablePro 采样项目以发现所有属性名称，并通过多数投票推断列类型。键属性（分区键、排序键）始终显示为前几列。

### PartiQL 查询

在编辑器中编写和执行 [PartiQL](https://docs.aws.amazon.com/amazondynamodb/latest/developerguide/ql-reference.html) 查询：

```sql
-- 查询项目
SELECT * FROM "Users" WHERE userId = 'user123'

-- 插入项目
INSERT INTO "Users" VALUE {'userId': 'user456', 'name': 'Alice', 'age': 30}

-- 更新项目
UPDATE "Users" SET name = 'Bob' WHERE userId = 'user123'

-- 删除项目
DELETE FROM "Users" WHERE userId = 'user123'
```

<Tip>
PartiQL 中的表名必须用双引号括起来：`"TableName"`。字符串值使用单引号：`'value'`。
</Tip>

### 数据类型

| DynamoDB 类型 | 显示 |
|---------------|------|
| **S**（String） | 纯文本 |
| **N**（Number） | 数值 |
| **B**（Binary） | Base64 编码字符串 |
| **BOOL** | `true` / `false` |
| **NULL** | `NULL` |
| **L**（List） | JSON 数组 |
| **M**（Map） | JSON 对象 |
| **SS**（String Set） | JSON 字符串数组 |
| **NS**（Number Set） | JSON 数字数组 |
| **BS**（Binary Set） | JSON Base64 字符串数组 |

### 数据编辑

直接在数据网格中编辑单元格值、插入行和删除行。更改自动生成 PartiQL 语句。

<Note>
键属性（分区键和排序键）不能修改。要更改键值，请删除项目并插入新项目。
</Note>

### 智能查询优化

当按分区键过滤且条件为等于时，TablePro 自动使用 Query API 而不是 Scan。Query 速度更快，消耗更少的读取容量。

### 容量和指标

表元数据显示计费模式、估计项目数、表大小和索引详情。

### 导出

将查询结果或表数据导出为 CSV、JSON、SQL 和其他格式。

IAM 用户或角色至少需要以下权限：

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [

        "dynamodb:DescribeTable",
        "dynamodb:Scan",
        "dynamodb:Query",
        "dynamodb:GetItem",

        "dynamodb:PartiQLUpdate",
        "dynamodb:PartiQLDelete"

      "Resource": "*"
    }
  ]
}
```

## 故障排除

### 认证失败

**症状**："Authentication failed" 或 "UnrecognizedClientException"

**解决方案**：

1. 验证 Access Key ID 和 Secret Access Key 是否正确
2. 检查凭证是否已过期或被轮换
3. 对于 AWS Profile，确认 `~/.aws/credentials` 中存在该配置文件
4. 对于 SSO，运行 `aws sso login` 刷新凭证

### 拒绝访问

**症状**："AccessDeniedException"

**解决方案**：

1. 检查 IAM 权限 - 用户/角色需要 DynamoDB 访问权限
2. 如果使用资源级权限，验证资源 ARN

### 区域不匹配

**症状**：找不到表或表列表为空

**解决方案**：

1. 确认选择了正确的 AWS 区域
2. DynamoDB 表是区域特定的 - `us-east-1` 的表在 `eu-west-1` 中不可见

## 已知限制

- **无持久连接。** DynamoDB 基于 HTTP，每个操作是独立的 API 请求。
- **无 SSH 或 SSL 配置。** DynamoDB 通过 URLSession 使用 HTTPS 传输，并使用自定义 SigV4 请求签名。
- **无模式编辑。** DynamoDB 不支持 ALTER TABLE。
- **项目数为估计值。** 项目计数由 DynamoDB 约每 6 小时更新一次。
- **基于游标的分页。** DynamoDB 内部使用续传令牌。
- **不支持 DAX。** DynamoDB Accelerator 不受支持。
